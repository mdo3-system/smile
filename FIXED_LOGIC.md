## 2. 見積時の受領図面（見積時PDF）と設計後図書の分離

### 見積時受領図面（`pdf_*` カテゴリ）
- `new_request.php` でのPDF提出時に登録される
- カテゴリ: `pdf_plan`, `pdf_elevation`, `pdf_layout`, `pdf_section`, `pdf_area_calc`
- **表示場所**: 左パネル「📋 見積時の受領図面」ボックス（管理者・依頼主の両方に表示）
- **編集不可**: 見積書と紐づいているため、後から変更しない

### 設計依頼後アップロード図書（`cad_*`, `app_doc`, `soil_*`, `spec_*` 等）
- 設計依頼（`request_design_start` action）時以降にアップロードされる
- **表示場所**: 中央パネル「依頼主アップロード図書」ボックス（`col_center_uploads.php`）
- **履歴管理**: バージョン管理あり（最新版を太字表示、過去版はプルダウン）
- **差し替え**: ファイル存在時は「差し替え理由（必須）」入力後にUP

## 3. 依頼種別と必要図書一覧

案件作成時に選択された依頼種別に応じて、以下の図書が必要となる。

### ① 許容応力度設計（req_permit=1）
**必要図書:**
- 意匠CADデータ（配置図, 1F/2F平面図, 立面図, 矩計図）
- 確認申請書（2面〜5面）
- 地盤調査報告書

**スケジュール目安:** 一次回答まで **12営業日**

### ② 性能表示壁量計算（req_wall=1）
**必要図書:**
- 意匠CADデータ（配置図, 1F/2F平面図, 立面図, 矩計図）
- 確認申請書（2面〜5面）
- 地盤調査報告書

**スケジュール目安:** 一次回答まで **7営業日**

### ③ 外皮計算（req_skin=1）
**必要図書:**
- 意匠CADデータ（配置図, 1F/2F平面図, 立面図, 矩計図）
- 仕様書（断熱材）
- サッシ・玄関ドア仕様書
- 24時間換気計算図書
- 設備機器カタログ

**スケジュール目安:** 一次回答まで **10営業日**

### ④ 天空率計算（req_sky=1）
**必要図書（見積内容に応じて判定・自動表示）:**
- 道路の資料（座標・測量図・道路台帳等）← `天空率 道路斜線` が見積に含まれる場合
- 真北の資料 ← `天空率 北側斜線` が見積に含まれる場合
- ※見積の `note` JSONに含まれる `name` フィールドを検査して自動判定する

**スケジュール目安:** 一次回答まで **10営業日**

### ⑤ 基礎・横架材許容応力度（req_opt_kisohari=1）
- 許容応力度設計（①）の図書に準じる
- **スケジュール目安:** 一次回答まで **12営業日**

### 補足図書（追加・差し替え時）
- 補正通知書（`correction_notice`）
- その他追加資料（`other_extra`）
- 地盤改良関連図書（`soil_impr`）← 地盤調査結果が「改良あり」の場合

## 4. 一次回答の要件・フロー

```
[見積依頼受付] → [管理者が見積シミュレーターで金額算出・PDF発行]
   ↓
[依頼主が見積確認・設計依頼ボタンから]
   ① 構造仕様を入力（木材の種類・壁の仕様等）
   ② 必要図書を個別スロットにアップロード
   ③ 「見積時から図面変更の有無」を選択・入力（必須）
   ↓ action: request_design_start
[ステータス: primary_prep に更新]
[チャットに「設計開始依頼」通知 + 管理者へメール]
   ↓
[管理者が図書を確認]
   ↓
[管理者が一次回答期日を設定]
   → action: set_primary_due_date
   → projects.primary_due_date を更新
   → ステータス: contracted に更新（primary_prep → contracted）
   → チャットに「スケジュール確定」通知
   ↓
[スケジュール表が全カラムで更新される]
```

## 5. スケジュール計算ロジック（営業日・玉突き方式）

`functions.php` の `addBusinessDays($dateStr, $days)` を使用。
**水曜・日曜をスキップ**して加算（土曜は稼働日扱い）。

| ステップ | 担当 | 計算基準 | 加算 |
|---------|------|---------|------|
| 設計図書の受領 | 依頼主 | 開始時 | - |
| 着手基準日（一次回答） | サポート | = `primary_due_date`（管理者設定） | - |
| 構造計算・図面 初回提示 | サポート | 一次回答から | +10営業日 |
| 構造図CB（内容確認） | 依頼主 | 初回提示から | +4営業日 |
| 修正図面UP | サポート | CB確認から | +3営業日 |
| 申請図書一式UP | サポート | 修正UPから | +3営業日 |
| 質疑・審査待機 | 確認機関 | 申請図書UPから | +30日（暦日） |
| 補正対応 | サポート | 質疑受領から | +7営業日 |
| 残金のご精算 | 依頼主 | 完了後 | +7日（暦日） |

### 実施日（実績）の玉突き
- 管理者は各ステップの「実施日」を入力できる（`update_schedule_actual` action）
- 実施日が入力されると、その日が次工程の**起算日**となる
- `projects.schedule_actuals` カラム（JSON型）に `{ ステップ番号: "YYYY-MM-DD" }` 形式で保存

## 6. 初期見積額の自動設定

- **タイミング**: 管理者が見積PDFを初回発行したとき（`api_save_estimate.php`）
- **条件**: `projects.initial_est_amount` が `0` または `NULL` の場合のみ上書き
- **保存先**: `projects.initial_est_amount`（税抜）、`projects.initial_est_date`（発行日）
- **依頼主画面での表示**: 「ご請求・お支払い状況」ボックスに表示
- **再発行時**: 2回目以降の見積発行では `initial_est_amount` は変更しない（`formal_est_amount` を使用）

### バージョン管理ルール
- `is_latest = 1`: 最新版（1つだけ）
- `is_latest = 0`: 旧バージョン（履歴として保持）
- 差し替え時は旧版を `is_latest = 0` に更新後、新版を `version + 1` でINSERT
- 差し替え理由は `update_reason` カラムに保存し、チャットへ自動投稿
