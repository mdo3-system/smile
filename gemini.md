# 構造設計サポート・ポータル システム仕様書

これまでの設計方針・仕様・実装の経緯をまとめたマスタードキュメントです。
本システムは、構造設計における「依頼主との仕様調整・図書受領・スケジュール管理」と「協力業者への作図発注・納品」を同一プラットフォーム上でフェーズごとに切り分けて一元管理するための業務管理ポータルです。

## 1. データベース構成（コアテーブル）

*   **`projects` (案件マスター)**
    *   `id`, `project_name`, `status` (quote_req / primary_prep / structural_dwg 等)
    *   `primary_due_date` (一次回答基準日)
    *   `schedule_actuals` (JSON: スケジュール各工程の「実施日(実績)」を保存し、玉突き計算の起点とする)
*   **`users` (ユーザー・業者管理)**
    *   `id`, `company_name`, `contact_name`, `role` (admin / client / subcontractor), `phone_number`
*   **`project_specs` (構造仕様)**
    *   木材・壁・金物などの詳細仕様や依頼時の申し送り事項をJSON等で保存。
*   **`project_files` (図書管理)**
    *   案件に紐づく各種図書（PDF/CADなど）。`file_category` (スロット別カテゴリ) と `is_latest` (最新フラグ) で履歴と種類を管理。Google DriveのファイルID(`drive_file_id`)で実態を参照。
*   **`messages` (チャット・通知)**
    *   依頼主と管理者のやり取りや、システムからの自動通知（見積完了、図書UP完了など）を保存。

## 2. システムの基本思想と業務フロー

1.  **フェーズの明確な分離（見積と設計依頼の分離）**
    *   入口は `new_request.php` で、「何の見積か（計算内容）」と「基本図面（平面・立面・配置・求積等）」のみをアップロードする非常にシンプルなUIとする。
    *   見積後、依頼主が `project_detail.php`（ダッシュボード）にて、適宜自分のタイミングで詳細仕様の入力や必要図書（CADデータ等）を追加アップロードする。
2.  **設計開始依頼とスケジュールの玉突き更新**
    *   必要図書がすべて揃った段階で、依頼主が「設計開始依頼」アクションを起こす。ここでステータスが `primary_prep`（一次回答準備中）に進む。
    *   管理者が「一次回答日」を設定するとスケジュールが確定。さらに各工程の「実施日（実績）」を入力すると、それ以降の期日が自動的に「玉突き計算」で再構築される。
3.  **図書の一元管理とGoogle Drive連携**
    *   全てのファイルアップロードは Google Drive API を通じてクラウドに保存され、スロット指定によりカテゴリ間違いによるバージョン破壊を防止する。

## 3. 現在の課題・進行中の開発（次期フェーズ計画）

1.  **通知の強化（SMS/Email連携）**: 見積依頼時やチャット更新時に、ダッシュボードを開かずとも依頼者が気付けるよう、SMSやメールに通知を飛ばす仕組み。
2.  **UI/UXの改善（モーダル化）**: 詳細画面(`project_detail.php`)における「設計依頼・図書UP」のアクションをモーダル化し、左パネルで「現在何が不足しているか」を可視化する。
3.  **詳細仕様・地盤調査の拡充**: 構造材の各部位ごとの材種指定や、地盤調査（調査済/未調査但し書/調査予定）の選択、地盤改良関連図書の複数ファイルUPへの対応。
4.  **契約・納期ステータスの整理**: 意匠CADデータがUPされた時点で「契約完了（納期未定）」となるフローの明確化。
